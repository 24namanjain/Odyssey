# Sync to Notion Workflow (MCP only)

When I invoke this, use the **Notion MCP plugin** directly. Do **not** run Python scripts, shell scripts, or write intermediate JSON payload files.

Bidirectional sync: **the side with the newer timestamp wins**. If content is identical, skip.

## Steps

1. **Find or create the root page**
   - Use `notion-search` for a page titled **Odyssey** (not the Odyssey database).
   - If no page exists, create one with `notion-create-pages` (icon: рҹ“ҡ).

2. **Build a reconcile plan**
   - Index all `docs/**/*.md` files (skip `docs/tags.md` and other files with unsupported HTML).
   - Walk the Notion page tree under Odyssey with `notion-fetch` (child `<page>` blocks = folders/files).
   - Match local paths to Notion pages by mirrored structure:
     - `docs/AWS/2-iam/l1-iam.md` вҶ’ Odyssey вҶ’ AWS вҶ’ 2-iam вҶ’ l1-iam
     - `docs/Docker/index.md` вҶ’ Odyssey вҶ’ Docker (folder index)
   - For each matched pair, compare timestamps:
     - **Local time:** prefer `git log -1` on the file; fall back to filesystem mtime.
     - **Notion time:** `last_edited_time` from `notion-fetch` metadata.
   - Decide action:
     - **Skip** вҖ” content already matches (no meaningful diff).
     - **Push** вҖ” local is newer, or page exists only locally.
     - **Pull** вҖ” Notion is newer, or page exists only in Notion.

3. **Push (local вҶ’ Notion)**
   - Strip YAML frontmatter from markdown before sending body content.
   - **New page:** `notion-create-pages` with `parent.page_id` set to the correct folder page (create missing folder pages first). Include an appropriate `icon` emoji.
   - **Existing page:** `notion-fetch` first, then `notion-update-page` with `replace_content`.
   - Preserve child pages when replacing content (include `<page url="...">` tags from fetch output if needed).

4. **Pull (Notion вҶ’ local)**
   - Convert fetched Notion markdown to plain markdown for the repo.
   - Write or update the matching file under `docs/`, creating parent directories as needed.
   - Preserve or add YAML frontmatter (`title`, `tags`) вҖ” do not wipe existing tags unless the page title changed.
   - For folder pages with children in Notion, use `index.md` in the matching local folder.

5. **Sync icons**
   - After content sync, set page icons via `notion-update-page` (`command: update_properties` + `icon`).
   - Use emoji that match section/topic (see mapping below).
   - Set `icon` on new pages at creation time via `notion-create-pages`.

6. **Report**
   - Summarize: pushed, pulled, skipped, failed (with reasons).

## Rules

- No `notion_sync/`, no `sync_notion.sh`, no `.mcp_*.json` payload files.
- No `NOTION_TOKEN` or `.env` required вҖ” MCP auth handles access.
- Read `notion://docs/enhanced-markdown-spec` before formatting or parsing page content.
- On timestamp ties, prefer **skip** if content matches; otherwise **push** local.
